# 📜 Code of Conduct & Core Project Rules

Welcome to the **H₂O Calculator 3000** ecosystem. To protect the integrity of our development pipeline, our mathematical parser accuracy, and our open-source workspace, all operators and contributors are required to adhere to the following rules.

---

## 🛠️ Section 1: The Core Architecture Guardrails

To maintain our lightweight, lightning-fast client-side execution framework, all incoming codebase modifications must pass these strict engineering filters:

1. **Absolute Zero-Dependency Policy:** We refuse to bloat our ecosystem. No `npm install`, no external framework integrations (React/Vue/Tailwind), and no runtime script CDNs. If a feature cannot be executed using vanilla modern ES6 JavaScript, native web browser layout APIs, and pure CSS, it will not be merged.
2. **Deterministic Mathematical Accuracy:** Our calculation engine demands total precision. Any changes modifying the tokenization or execution loops must include updated validation tests protecting against floating-point errors and operator order logic failures.
3. **Pristine Presentation Layers:** Our neon glassmorphic design and mobile responsive grid are core features. Contributions must not degrade performance, break responsive scaling on microviewports, or alter the color styling variables without layout validation.

---

## 👥 Section 2: Community Behavior Standards

We are building a highly focused space for math enthusiasts and developers. To ensure collaboration remains productive, please observe the following behavioral rules:

* **Empathy over Ego:** Treat incoming developers and operators with respect. Help newcomers configure their local server environments or master Git clone mechanics without being condescending.
* **Constructive Criticism Only:** Code reviews should be direct, logical, and focused entirely on architectural improvement. Address code patterns, not personal competence.
* **Keep the Issues Queue Operational:** Do not use the bug tracking system for chat, feature brainstorming, or casual questions. Use **GitHub Discussions** for community dialogue, and reserve **Issues** for verified architectural breakdowns using our designated templates.

---

## 🚨 Section 3: Enforcement Strategy

Violations of these project rules—whether introducing unauthorized dependencies, breaking communication standards in PR reviews, or spamming the issue queues—will be addressed immediately by the repository administrative team (`afifashamsadvivo`). 

* **First Event:** A formal direct warning on the tracking thread and remediation request.
* **Second Event:** Immediate closure of active PRs/Issues and a temporary block from the repository matrix.
* **Severe Misconduct:** Permanent block from contributing to the organization namespace.

*Thank you for helping us keep the H₂O Calculator 3000 matrix pristine, precise, and fast!*
